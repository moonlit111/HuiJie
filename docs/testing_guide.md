# 慧杰 Mod 完整测试指南

---

## 0. 测试前准备

### 安装步骤
1. 确保已安装 **SMAPI 4.0+** 和 **Content Patcher 2.0+**
2. 将 `HuiJie` 文件夹复制到 Stardew Valley 的 `Mods/` 目录
3. 启动游戏，进入已有存档或新建存档

### SMAPI 日志位置
- **macOS**: `~/.config/StardewValley/ErrorLogs/SMAPI-latest.txt`
- **Windows**: `%appdata%/StardewValley/ErrorLogs/SMAPI-latest.txt`
- **Linux**: `~/.config/StardewValley/ErrorLogs/SMAPI-latest.txt`

### 调试命令（在游戏内按 `/` 输入）
| 命令 | 用途 |
|------|------|
| `/debug huijie` | 如果NPC未出现，查看相关日志 |
| `/friendship huijie X` | 设置好感度（X = 心数×250，如2心=500） |
| `/time HHMM` | 设置当前时间（如 `/time 2100` 设为21:00） |
| `/season spring/summer/fall/winter` | 切换季节 |
| `/day X` | 设置日期 |
| `/weather rain/sun/storm` | 切换天气 |
| `/warp Farm` | 传送到农场 |
| `/dialogue huijie` | 触发慧杰对话 |

> 推荐使用 SMAPI 的 **ChestsAnywhere** 或 **DebugMode** 模组辅助测试。

## 1. 基础加载测试

### 1.1 SMAPI 加载检查

**启动游戏后**，打开 SMAPI 控制台（或日志文件），确认以下内容：

```
✅ 检查项：
[Content Patcher] Loaded asset 'Characters/huijie'          → 精灵图加载
[Content Patcher] Loaded asset 'Portraits/huijie'           → 肖像图加载
[Content Patcher] Loaded asset 'Data/Characters/huijie'     → NPC定义加载
```

**如果任何一项未出现**：
- 检查 `content.json` 中的 `FromFile` 路径是否正确
- 检查 `manifest.json` 中的 `UniqueID` 是否唯一
- 确认 Content Patcher 版本 ≥ 2.0

### 1.2 NPC 出现测试

1. 进入游戏（已有存档需**睡一天**让 dayUpdate 加载日程）
2. 前往 **格斯酒馆（Saloon）** 二楼
3. 确认慧杰在坐标 (35, 8) 附近出现

**如果 NPC 不出现**：
- 睡一天后重试（新NPC在已有存档需要 dayUpdate）
- 查看 SMAPI 日志是否有错误

## 12. 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| NPC不出现 | 新存档需dayUpdate | 睡一天 |
| NPC不移动 | 日程未加载 | 睡一天；检查SMAPI日志中`Characters/schedules/huijie` |
| 对话是默认文本 | EditData未生效 | 检查content.json的Include路径 |
| 心事件不触发 | 条件不满足 | 检查时间、天气、好感度、是否已婚 |
| 动画不播放 | 动画定义错误 | 检查animations.json的帧号和速度 |
| 特殊任务不出现 | 季节/心数不满足 | 检查RequiredTags和CP When条件 |
| 信件不寄出 | 触发事件ID冲突 | 检查customMail.json中的事件ID唯一性 |
| SMAPI报错JSON | 语法错误 | 检查尾逗号、引号匹配、注释格式 |
